The dialogue module gives you "script tree authoring + runtime progression": designers draw nodes and options in the dashboard; players traverse branches on the client; each node entry can award rewards.

## Data model

- **dialogue_scripts**: full script tree — name, starting node, all nodes (text, options list, reward config).
- **dialogue_progress**: per-player × per-script progress — `currentNodeId` / `visitedNodeIds` / `claimedRewards`.

## Admin API

```http
POST   /api/dialogue/scripts
GET    /api/dialogue/scripts
PATCH  /api/dialogue/scripts/{id}
DELETE /api/dialogue/scripts/{id}
```

## Client API

```http
POST /api/dialogue/end-user/{userId}/start        body: { scriptId }      begin or reset
POST /api/dialogue/end-user/{userId}/choose       body: { optionId }      pick a branch
GET  /api/dialogue/end-user/{userId}/progress/{scriptId}
```

## Business rules

- **Tree structure**: each node has text + N options; each option points to a next `nodeId` or carries an action (link to external URL, trigger another script, end).
- **On-enter rewards**: `node.onEnter.reward`, if set, dispenses on entry. The service uses `item_grant_logs` `ON CONFLICT` to guarantee the same `(user, script, node)` rewards once only.
- **Option conditions**: options may have a `condition` (player level / collection-entry unlocked / holds an item). Unmet conditions gray out.
- **Reset**: calling `start` on the same script returns to the start node; previous reward records are preserved (no double-rewards).

## Combining with other modules

- [item](./item): node rewards flow through the item service.
- [level](./level): option conditions can reference the player's progress in the level module.
- [link](./link): option `action` uses the link value object to express jump destinations.

## Events

`dialogue.started` / `dialogue.node_entered` / `dialogue.reward_granted` / `dialogue.finished`.
