The guild module models "hall-type social organizations": players create or join guilds, roles split into leader / officer / member, guilds can level up, gate joining, and track member contributions.

## Data model

- **guild_settings** (one row per org): guild-creation prerequisites, baseline max members, level-up rules, default join mode.
- **guild_guilds**: a guild's basic info — name / tag / leader / level / exp / joinMode.
- **guild_members**: unique on `(guildId, endUserId)`, with role (leader / officer / member), contribution score, join time.
- **guild_join_requests**: application flow (only when `joinMode=request`).
- **guild_contribution_logs**: atomic log of each contribution (fighting, gifting, activities) — the source of truth for level-up exp.

## Admin API

```http
PUT    /api/guild/settings
POST   /api/guild/guilds                   admin-created guilds
GET    /api/guild/guilds
PATCH  /api/guild/guilds/{id}
DELETE /api/guild/guilds/{id}
POST   /api/guild/guilds/{id}/transfer-leader
```

## Client API

```http
POST   /api/guild/end-user/create          player creates
POST   /api/guild/end-user/{id}/join
POST   /api/guild/end-user/{id}/request
POST   /api/guild/end-user/{id}/approve/{requestId}
POST   /api/guild/end-user/{id}/leave
POST   /api/guild/end-user/{id}/contribute body: { amount, reason }
GET    /api/guild/end-user/{id}/members
```

## Business rules

- **joinMode**: `open` (instant) / `request` (approval needed) / `closed` (invite-only).
- **Member cap**: baseline from settings, multiplied by `level_rules` per guild level.
- **Level / exp**: `contribute` atomically updates `member.contribution` + `guild.exp` + checks the level-up threshold, using `xmax=0` serialization for concurrent contributions.
- **Transfer leader**: `transfer-leader` is a single atomic swap — no split-brain.
- **Leave / kick**: if the leader leaves, either transfer or disband (configurable).

## Combining with other modules

- [leaderboard](./leaderboard): `scope=guild` for intra-guild contribution boards.
- [activity](./activity): guild-scoped activities; the guild service provides membership checks.
- [mail](./mail): guild-wide broadcast mails, using the guild member list as recipients.
- `chat`: future guild channel (chat module).

## Events

`guild.created` / `guild.joined` / `guild.left` / `guild.level_up` / `guild.disbanded`.
