The collection module powers "Pokedex / weapon codex / monster compendium" — collection-based unlock progression: configure an album, players unlock entries via various paths, and crossing thresholds triggers milestone rewards.

## Data model

- **collection_albums**: album metadata — alias, group definitions, total entries.
- **collection_entries**: an entry (card / monster) inside an album, with an unlock-condition key.
- **collection_milestones**: milestone — scope `entry` / `group` / `album`, threshold triggers rewards.
- **collection_user_entries**: per-player × per-entry unlock state.

## Admin API

```http
POST   /api/collection/albums
GET    /api/collection/albums
PATCH  /api/collection/albums/{key}
DELETE /api/collection/albums/{key}
POST   /api/collection/albums/{key}/entries
POST   /api/collection/albums/{key}/milestones
```

## Client API

```http
POST /api/collection/end-user/{userId}/album/{key}/unlock      body: { entryKey }
POST /api/collection/end-user/{userId}/album/{key}/claim       body: { milestoneKey }
GET  /api/collection/end-user/{userId}/album/{key}
```

## Business rules

- **Idempotent unlock**: the same `(userId, entryKey)` unlocks once only — `ON CONFLICT DO NOTHING`.
- **Milestone scope**:
  - `entry`: a specific entry was unlocked (e.g. unlocking "Legendary Pokemon" triggers rewards).
  - `group`: X entries in a group unlocked.
  - `album`: Y entries across the whole album unlocked.
- **Auto-claim**: with `autoClaim=true`, the milestone auto-dispenses rewards (via [item](./item) + [mail](./mail)) the moment it's achieved — no player action needed.
- **Reward pipeline**: `claim` atomically validates the condition + calls `itemService.grantItems` + marks claimed.

## Combining with other modules

- [item](./item): unlock rewards flow from here.
- [mail](./mail): auto-claim rewards land via mail.
- [activity](./activity): activity-scoped special albums that archive when the activity ends.

## Events

`collection.unlocked` / `collection.milestone_claimed`.
