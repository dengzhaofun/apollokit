> This page is for engineering teams. For product background and use cases, see the [Product Overview](./index).

Storage Box is a **currency deposit + interest** module. Players deposit [currency](../currency) into a box; interest accrues per the box's rules; on maturity (or early withdrawal) rewards/penalties settle back to the wallet.

## Data model

- **config**: currency, min/max deposit, period, interest rate (or tiered), early-withdrawal penalty.
- **deposit**: a single deposit — principal, start time, maturity time, interest accrued so far.
- **withdrawal**: principal + interest + penalty record on withdrawal.

## Admin API

```http
POST   /api/storage-box/configs             create
GET    /api/storage-box/configs
GET    /api/storage-box/configs/{id}
PATCH  /api/storage-box/configs/{id}
DELETE /api/storage-box/configs/{id}        cascades deposits under this box

POST   /api/storage-box/deposits            deposit on behalf of a player (ops / support)
POST   /api/storage-box/withdrawals         withdraw on behalf
GET    /api/storage-box/deposits/{endUserId}   list active deposits with projected interest
```

The public client API shape (`/api/client/storage-box/*`) is still evolving. Check `/openapi.json` for the current shape.

## Interest calculation

A pure function `computeInterest` (see `modules/storage-box/interest.ts`) takes deposit time, current time, the configured annual / period rate, and the early-withdrawal flag, and returns:

```ts
{
  principal: number;  // principal
  interest: number;   // accrued interest (can be negative — early-withdrawal penalty)
  penalty: number;    // penalty (if any)
  payout: number;     // final = principal + interest - penalty
}
```

The function is pure — both the read path and the withdrawal path use it, so `GET /deposits/{endUserId}`'s `projectedInterest` stays consistent with actual withdrawal (assuming time hasn't shifted).

## Atomic settlement

Withdrawal is a single SQL statement:

1. Validate the deposit exists and hasn't been withdrawn.
2. Compute payout.
3. Mark deposit `withdrawn`.
4. Grant payout to the player's wallet.
5. Write withdrawal row.

All in one SQL, avoiding split-brain under concurrency.

## Gotchas

- Can't transfer between configs — to "switch boxes", withdraw then deposit.
- Rate changes after deposit **don't apply** to existing deposits (the deposit row snapshots the rate at deposit time).
- No partial withdrawal. To withdraw part, deposit in two amounts.
