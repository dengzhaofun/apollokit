物品模块是整个平台的基础资源层，管理虚拟物品的定义、分类和用户持有。签到奖励和兑换系统均通过物品模块执行实际的资源变动。

## 核心概念

### 物品分类 (ItemCategory)

物品的逻辑分组，如"货币"、"道具"、"材料"：

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 分类名称 |
| `alias` | string? | 组织内唯一别名 |
| `icon` | string? | 图标 URL |
| `sortOrder` | number | 排序权重 |
| `isActive` | boolean | 是否启用 |

### 物品定义 (ItemDefinition)

定义一种物品的属性和堆叠规则：

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 物品名称 |
| `alias` | string? | 组织内唯一别名（如 `gold`、`gem`） |
| `categoryId` | UUID? | 所属分类 |
| `description` | string? | 描述 |
| `icon` | string? | 图标 URL |
| `stackable` | boolean | 是否可堆叠（默认 `true`） |
| `stackLimit` | number? | 单堆上限（`null` = 无限） |
| `holdLimit` | number? | 用户持有上限（`1` = 唯一物品） |
| `isActive` | boolean | 是否启用 |

### 堆叠规则

物品在背包中的存储方式取决于堆叠设置：

| 场景 | 行为 |
|------|------|
| `stackable=true`, `stackLimit=null` | 无限堆叠，每个 (用户, 物品) 一行记录（适合货币） |
| `stackable=true`, `stackLimit=99` | 有限堆叠，超过 99 自动分堆 |
| `stackable=false` | 不可堆叠，每个实例独立一行（数量恒为 1） |

### 背包视图 (InventoryView)

查询背包时返回聚合视图：

```json
{
  "definitionId": "uuid",
  "definitionAlias": "gold",
  "definitionName": "金币",
  "icon": "https://...",
  "stackable": true,
  "totalQuantity": 150,
  "stacks": [
    { "id": "stack-uuid-1", "quantity": 99, "instanceData": null },
    { "id": "stack-uuid-2", "quantity": 51, "instanceData": null }
  ]
}
```

### 审计日志 (ItemGrantLog)

每次发放/扣除都会记录审计日志：

| 字段 | 说明 |
|------|------|
| `delta` | 变动量（正数=发放，负数=扣除） |
| `source` | 来源标识（如 `check_in`、`exchange`、`admin_grant`） |
| `sourceId` | 幂等键 |
| `quantityBefore` | 变动前数量 |
| `quantityAfter` | 变动后数量 |

## 管理端 API

### 分类 CRUD

```http
POST   /api/item/categories              # 创建分类
GET    /api/item/categories              # 列出分类
GET    /api/item/categories/{key}        # 按 UUID 或 alias 查询
PATCH  /api/item/categories/{id}         # 更新分类
DELETE /api/item/categories/{id}         # 删除分类
```

### 物品定义 CRUD

```http
POST   /api/item/definitions             # 创建物品定义
GET    /api/item/definitions             # 列出物品定义
GET    /api/item/definitions/{key}       # 按 UUID 或 alias 查询
PATCH  /api/item/definitions/{id}        # 更新物品定义
DELETE /api/item/definitions/{id}        # 删除物品定义
```

创建物品定义示例：

```json
{
  "name": "金币",
  "alias": "gold",
  "categoryId": "currency-category-uuid",
  "description": "通用货币",
  "stackable": true,
  "stackLimit": null,
  "holdLimit": null,
  "isActive": true
}
```

### 发放物品

```http
POST /api/item/grant
Authorization: Bearer ak_...
Content-Type: application/json

{
  "endUserId": "user-42",
  "grants": [
    { "definitionId": "gold-uuid", "quantity": 100 },
    { "definitionId": "gem-uuid", "quantity": 10 }
  ],
  "source": "admin_grant",
  "sourceId": "optional-idempotency-key"
}
```

**响应**:

```json
{
  "grants": [
    {
      "definitionId": "gold-uuid",
      "quantityBefore": 50,
      "quantityAfter": 150,
      "delta": 100
    },
    {
      "definitionId": "gem-uuid",
      "quantityBefore": 0,
      "quantityAfter": 10,
      "delta": 10
    }
  ]
}
```

### 扣除物品

```http
POST /api/item/deduct
Content-Type: application/json

{
  "endUserId": "user-42",
  "deductions": [
    { "definitionId": "gold-uuid", "quantity": 50 }
  ],
  "source": "exchange",
  "sourceId": "exchange-uuid"
}
```

扣除时如果用户持有量不足，返回错误。

### 查询背包

```http
GET /api/item/users/{endUserId}/inventory
GET /api/item/users/{endUserId}/inventory?definitionId=uuid   # 按物品过滤
```

### 查询余额

```http
GET /api/item/users/{endUserId}/balance/{key}
```

`key` 支持物品定义的 UUID 或 alias。

**响应**:

```json
{
  "definitionId": "gold-uuid",
  "balance": 150
}
```

## 客户端 API

### 查询背包

```http
GET /api/client/item/users/{endUserId}/inventory
x-api-key: cpk_...
x-user-hash: hmac_hex_string
```

支持 `definitionId` 查询参数过滤。

### 查询余额

```http
GET /api/client/item/users/{endUserId}/balance/{key}
x-api-key: cpk_...
x-user-hash: hmac_hex_string
```

> 客户端 API 仅支持查询，不支持直接发放/扣除。物品变动只能通过管理端 API 或签到/兑换系统间接触发。

## holdLimit 行为

当物品定义设置了 `holdLimit` 时：

- 发放时，如果发放后总量超过 `holdLimit`，实际发放量会被截断
- 例：`holdLimit=100`，用户当前持有 90，发放 20 → 实际只发放 10
- `holdLimit=1` 适用于唯一物品（如成就徽章），重复发放不会增加数量
