ApolloKit 是一个面向游戏的 SaaS 平台，提供签到、兑换、物品等通用模块的 API。本文档面向**对接方开发者**，帮助你快速接入各模块功能。

## 架构概览

- **API 框架**: Hono + Zod OpenAPI（运行在 Cloudflare Workers 上）
- **数据库**: Neon Postgres（通过 Drizzle ORM）
- **认证**: Better Auth（管理端）+ Client Credentials（客户端）
- **文档**: OpenAPI 规范，可在 `/docs`（Scalar UI）和 `/openapi.json` 获取

## 模块列表

| 模块 | 路径前缀 | 说明 |
|------|---------|------|
| [认证与凭证](/docs/authentication) | `/api/auth/*`, `/api/client-credentials/*` | 管理端认证、API 密钥、客户端凭证与 HMAC 验证 |
| [签到系统](/docs/check-in) | `/api/check-in/*` | 每日签到、连续签到追踪、周期重置、签到奖励 |
| [兑换系统](/docs/exchange) | `/api/exchange/*` | 积分/物品兑换、用户/全局限额、幂等执行 |
| [物品系统](/docs/item) | `/api/item/*` | 物品定义与分类、用户背包、发放与扣除、审计日志 |
| [TypeScript SDK](/docs/sdk) | — | 类型安全的管理端 SDK，从 OpenAPI 规范自动生成 |

## 两种用户身份

ApolloKit 区分两种用户概念：

- **管理端用户 (Admin User)**: 通过 Better Auth 认证的 SaaS 运营方，拥有组织 (Organization) 上下文
- **终端用户 (End User)**: 对接方业务系统中的用户，以 `endUserId`（不透明字符串）标识，ApolloKit 不管理其身份

所有业务数据按 `organizationId` 隔离，终端用户操作需通过 Client Credentials 认证。

## 错误响应格式

所有错误返回统一结构：

```json
{
  "error": "描述信息",
  "code": "ERROR_CODE",
  "requestId": "请求追踪 ID"
}
```

常见错误码：`NOT_FOUND`、`UNAUTHORIZED`、`FORBIDDEN`、`CONFLICT`、`INVALID_INPUT`、`LIMIT_EXCEEDED`
