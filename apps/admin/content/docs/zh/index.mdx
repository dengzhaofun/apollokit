ApolloKit 是面向游戏 / 应用的 **SaaS 后端平台**——把「每款游戏都要重写一遍」的通用逻辑(经济系统、运营活动、社交玩法、埋点)变成可配置的模块 + 类型安全的 SDK。本文档面向**接入方开发者**,帮你从零到跑通第一次调用、再到对接全部 30+ 模块。

不知从哪开始?先看 [快速开始](./quickstart),5 分钟能跑通第一次端到端调用。

## 两类用户身份

接入前务必区分清楚:

- **管理端用户(Admin User)**:通过 Better Auth 登录管理台的 SaaS 运营方。拥有 `organizationId` 上下文(产品语义即「项目」)。
- **终端用户(End User)**:接入方业务系统里的玩家,以不透明的 `endUserId` 标识。ApolloKit **不**管它的身份系统——你把 id 传进来就行。

详细模型见 [项目与多租户](./projects) 与 [认证与凭证](./authentication)。

## 统一错误体

所有失败都返回:

```json
{
  "error": "human readable",
  "code": "module.reason",
  "requestId": "req_01hx..."
}
```

按 `code` 做程序分支,按 `requestId` 查日志。完整清单见 [errors](./errors)。
