图鉴模块实现「宝可梦图鉴 / 武器图鉴 / 怪物图鉴」这种收集式解锁进度玩法：管理员配置一整本图鉴，玩家通过各种途径解锁条目，积累到里程碑阈值自动触发奖励。

## 解决什么问题

「收集全部」是游戏中最强的长线留存钩子之一，但每次都要重写收集系统：

- 如何记录每个玩家对每个条目的解锁状态（数据量大）
- 里程碑是「全部收集完」才触发，还是「任意 5 个分类」都算
- 玩家主动领奖 vs 自动发奖，要不要防重复

图鉴模块把这些规则全部封装好，配置一本图鉴就能直接使用。

## 典型使用场景

**场景 1：武器图鉴**
100 种武器，每把武器首次获得自动解锁图鉴条目。集齐 10 把发一个奖励，集齐 50 把发大奖，集齐全部发终极皮肤。

**场景 2：怪物图鉴**
击杀某只怪物首次解锁该条目。按「地区」分组，集齐某个地区所有怪物触发地区里程碑奖励。

**场景 3：活动限定图鉴**
活动期间限定收集册，活动结束后归档。玩家在活动期间尽量收集，活动结束时快照留存。

**场景 4：成就驱动的图鉴**
「首次组队」「首次购买」「首次满星」等一次性成就，通过图鉴统一管理，不用在各模块散写。

## 核心能力

- **分组管理**：图鉴内支持按分类分组，里程碑可针对整册或单个分组设置
- **多种里程碑触发**：整册进度 / 分组进度 / 特定条目解锁，灵活配置
- **自动发奖**：里程碑支持 `autoClaim`，达成即自动发奖，无需玩家主动领取
- **幂等解锁**：同一条目多次解锁只记一次，安全可重试

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 图鉴是成本最低的长线留存机制之一 |
| 产品经理 | 灵活配置收集规则和里程碑奖励，适配不同内容体量 |
| 运营人员 | 查看集齐率数据，辅助下一版本内容规划 |
| 研发团队 | 在玩家获得新内容时调一行解锁接口，其余自动处理 |
