素材云盘是管理台内置的通用资源库：策划可以上传图片、音频、JSON 配置文件，在文件夹树中整理分类，其他模块（banner、邮件、实体等）可以直接引用这里的资源 URL，无需每次走研发上传。

## 解决什么问题

游戏运营中有大量需要频繁更换的素材：

- 活动 banner 图每次活动都要换一张，但上传图片要找研发 / 走 CDN 后台
- 实体的图标、道具的贴图、对话的背景图分散在不同地方，没有统一管理入口
- 素材变更需要发版或联系研发，运营完全不能自助

素材云盘让运营直接在管理台上传和管理所有素材，其他模块直接引用，研发只需对接一次。

## 典型使用场景

**场景 1：活动 banner 素材管理**
运营在「活动 / banner 素材」文件夹上传新活动的海报图，复制 URL 填入 banner 配置——全程不需要研发介入。

**场景 2：实体/道具图标**
游戏新增英雄，策划将英雄头像上传到「实体 / 英雄」文件夹，在 blueprint 配置里填入图片 URL 即可展示。

**场景 3：JSON 配置文件**
游戏平衡性调整涉及的数值配置 JSON 上传到云盘，游戏客户端定期拉取最新版本，不需要重新发版。

**场景 4：邮件附图**
运营发送带图片的游戏邮件，将图片上传到云盘后，在邮件正文中引用 URL 即可显示内嵌图片。

## 核心能力

- **文件夹树管理**：多级文件夹结构，按业务场景分类整理
- **直传到 R2/S3**：前端直接 PUT 到对象存储，不经过业务服务器，大文件友好
- **公共代理端点**：统一的代理 URL 供客户端访问，内置 CDN 缓存
- **租户隔离**：不同项目的素材物理隔离，互不可见
- **大小限制**：单文件最大 10 MB，可按需调整

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 运营团队资源管理完全自助，减少研发依赖 |
| 运营人员 | 上传 → 复制 URL → 填到配置里，全程自助 |
| 策划 | 素材集中管理，告别「图片丢了/找不到」的问题 |
| 研发团队 | 对接一次上传流程，后续所有素材变更无需介入 |
