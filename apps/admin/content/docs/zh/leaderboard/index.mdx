排行榜模块将「按某个指标排名」抽象成通用组件：自定义指标、多种聚合方式、按周期自动结算留存快照，支持全局/公会/好友等多种范围分榜。

## 解决什么问题

排行榜是游戏里最常见的竞争机制，但每次都要从头做：

- 怎么处理并发投分（两个玩家同时打了高分，数据库怎么保持正确）
- 周期结算后历史记录要不要保留
- 好友榜和全服榜是同一份数据，还是要分别存
- 周期重置时如何平滑切换，不影响正在进行的游戏

排行榜模块把这些通用问题封装好，业务侧只管「投分」，剩下的自动完成。

## 典型使用场景

**场景 1：本周累计伤害榜**
玩家每场战斗结束后，业务上报本场伤害量，排行榜按 `sum` 聚合，显示本周累计伤害排名。周一自动结算上周快照，新的一周重新开始。

**场景 2：公会贡献榜**
公会内成员按贡献值排名，只显示本公会成员，`scope=guild`。激励成员积极参与公会活动。

**场景 3：好友积分比拼**
我的朋友中谁积分最高？`scope=friend` 只拉好友的排名，做成「好友积分比拼」功能，增加社交互动。

**场景 4：历史赛季查询**
本赛季结束后，玩家可以查看历史赛季的最终排名快照。数据永久留存，用于赛季奖励发放依据。

## 核心能力

- **自定义指标**：任意业务指标都可以建榜，不限模块
- **多种聚合方式**：sum（累加）/ max（取最高）/ latest（取最新），覆盖不同排名语义
- **多范围分榜**：全局、公会、好友、活动，一次投分自动命中多个榜
- **周期管理**：日 / 周 / 月 / 全时，到期自动生成结算快照
- **幂等投分**：调用方带幂等键，防止同一事件重复计入

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 排行榜是驱动竞争、延长游戏时长的核心机制 |
| 产品经理 | 快速建立各种维度的排名，支持活动专属榜、赛季榜等复杂需求 |
| 运营人员 | 查看实时排行数据，设计差异化奖励，配置结算时间 |
| 研发团队 | 一次接入多个榜，并发安全由模块保证 |

## 与其他模块的关系

- **[任务系统](../task)**：任务完成后投分到排行榜，做「本周任务积分榜」
- **[活动](../activity)**：活动专属榜，以活动 id 作为周期 key
- **[公会](../guild)**：公会内贡献榜，`scope=guild`
- **[天梯](../rank)**：竞技天梯请用天梯模块——天梯有 Elo/段位/赛季完整机制；排行榜适合非匹配的积分比拼
