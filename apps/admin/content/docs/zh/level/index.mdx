关卡模块实现「章节 → 关卡 → 星级」的结构化关卡系统：关卡有解锁规则，通关带星，满星有额外奖励，所有进度在服务端持久化，不依赖客户端本地存档。

## 解决什么问题

大多数游戏都有线性内容推进，但自己实现关卡系统会遇到：

- 关卡解锁条件多样（通过上一关 / 累积星数 / 章节通关），每种都要特判
- 星级是累加还是取最高？（玩家重玩拿了 2 星，原来 3 星要不要降）
- 奖励领取的幂等性——同一颗星的奖励不能领两次

关卡模块把这些规则标准化，策划在管理台配置，客户端只需上报通关结果。

## 典型使用场景

**场景 1：线性章节推进**
10 章共 100 关，第 2 章的第一关要求第 1 章全部通关才解锁。玩家必须按顺序解锁，服务端自动判定解锁资格。

**场景 2：星级额外奖励**
每关最多 3 星，1 星通关给基础奖励，3 星通关给额外稀有道具。奖励独立领取，玩家可以先 1 星通关再回来刷 3 星。

**场景 3：解锁条件多样化**
某个隐藏关卡需要全图 200 颗星才能解锁，另一个关卡需要通过特定关卡才能进入，通过配置支持多种解锁规则。

**场景 4：活动专属关卡**
活动期间限定关卡，活动结束后自动归档，不影响主线关卡进度。

## 核心能力

- **章节管理**：多章节结构，章节有独立的解锁条件
- **灵活解锁规则**：自动开放 / 通过特定关 / 累积星数 / 章节通关，配置驱动
- **星级系统**：每关最多 N 星（可配置），星级取历史最高，不因重玩降星
- **分星奖励**：每颗星可独立配置奖励，独立领取，幂等防重复
- **服务端存档**：所有进度在服务端持久化，多端同步，不依赖本地存档

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 线性内容进度集中管理，防止存档丢失引发的客诉 |
| 产品经理 | 灵活配置解锁条件和奖励结构，支持复杂关卡设计 |
| 策划 / 运营 | 管理台直接配置关卡属性和奖励，实时生效 |
| 研发团队 | 上报通关结果即可，解锁判定和奖励发放由模块处理 |
