实体模块是一个「面向角色 / 怪物 / 装备通用数据的建模层」。把游戏里「有属性、可升级、可换皮、可编队」的单位全部抽象成 `entity`——用 schema 定义属性字段,用 blueprint 定义具体模板,用 skin 叠 buff,用 formation 约束阵容。

## 数据模型

- **entity_schemas**:定义属性字段(HP / ATK / DEF ...)、标签(如 `flying`、`undead`)、资源槽。一个 schema 是「这一类单位的形状」。
- **entity_blueprints**:具象模板——一个 `pikachu` 蓝图引用 `creature` schema,填字段值。
- **entity_blueprint_skins**:给 blueprint 挂皮肤,皮肤可叠加属性 bonus。
- **entity_instances**:玩家实际拥有的实体实例(带等级、已上阵等状态)。
- **entity_formations**:阵容配置——N 个槽位、每槽接受什么类型、是否允许重复。

## 管理端 API

```http
POST   /api/entity/schemas
POST   /api/entity/blueprints
POST   /api/entity/blueprints/{id}/skins
POST   /api/entity/formations              定义阵容模板
```

## 客户端 API

```http
POST /api/entity/end-user/acquire          body: { blueprintId }    获得一个实体实例
POST /api/entity/end-user/equip-skin       body: { instanceId, skinId }
POST /api/entity/end-user/formation/save   body: { formationId, slots }
GET  /api/entity/end-user/instances
```

## 业务规则

- **继承关系**:blueprint 的属性 = schema 默认 + blueprint 覆盖;instance 的属性 = blueprint + 等级加成 + 当前 skin bonus。分层计算,不在数据库做 JOIN 运算。
- **阵容校验**:`formation/save` 时校验每个槽位的实体类型是否匹配、是否超出重复上限。
- **删除限制**:blueprint 有在用的 instance 时不能硬删,只能标 archived。

## 与其他模块组合

- [level](./level):实体升级曲线可由 level 管理,也可直接用 entity 内部的经验字段。
- [item](./item):获得实体 = 发一个 item,item 后处理器调 `entity/end-user/acquire`。
- [collection](./collection):拿到新 blueprint 的 instance → 解锁图鉴条目。
- [rank](./rank):阵容与天梯结算无关——rank 只看结果,阵容由客户端上报。

## 事件

`entity.acquired` / `entity.skin_equipped` / `entity.formation_saved`。
