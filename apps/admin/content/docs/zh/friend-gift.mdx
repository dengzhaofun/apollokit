好友赠礼模块实现「像《王者荣耀》早期那种好友间互送友情点」的日常互动:管理员预设礼包种类,玩家按日限额互送,收方一键领取。

## 数据模型

- **friend_gift_settings**(每 org 一行):每日发送 / 领取上限、有效期、时区。
- **friend_gift_packages**:可送礼包模板——名称、内容物、CD。
- **friend_gift_sends**:一次发送记录(sender → receiver + package + status)。
- **friend_gift_daily_states**:玩家的每日发送 / 领取计数器,按 `(endUserId, date)` 联合键。

## 管理端 API

```http
PUT  /api/friend-gift/settings
POST /api/friend-gift/packages
GET  /api/friend-gift/packages
PATCH /api/friend-gift/packages/{id}
DELETE /api/friend-gift/packages/{id}
```

## 客户端 API

```http
POST /api/friend-gift/end-user/send        body: { receiverId, packageId }
POST /api/friend-gift/end-user/claim/{sendId}
POST /api/friend-gift/end-user/cancel/{sendId}

GET  /api/friend-gift/end-user/inbox       收到的
GET  /api/friend-gift/end-user/outbox      发出去的
GET  /api/friend-gift/end-user/daily-state 今日剩余次数
```

## 业务规则

- **双向日限**:送方有「每日最多送 N 次 / 每人 1 次」、收方有「每日最多领 M 次」。两侧任一超限即拒。
- **状态流**:`pending` → `claimed`(领了) / `expired`(过了 TTL 没领) / `cancelled`(发送方撤回)。
- **过期自动清理**:每分钟 cron 扫过期的 pending 送礼,转 expired。
- **领取发奖**:`claim` 原子地读 package 内容、调 [item](./item) 服务发放、标 send 为 claimed。
- **时区**:「每日」以 settings 里的时区计算边界,不是 UTC。

## 与其他模块组合

- [friend](./friend):送礼的前提是双方是好友(可选 setting 要求)。
- [item](./item):礼包内容走 item 服务发放。
- [mail](./mail):玩家没上线时可把未领礼物转邮件(可选)。

## 事件

`friend-gift.sent` / `friend-gift.claimed` / `friend-gift.expired`。
