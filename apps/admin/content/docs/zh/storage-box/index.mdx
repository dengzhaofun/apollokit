存储箱是一个**货币存款 + 计息结算**的模块。玩家把游戏内货币存进一个「箱子」，按箱子配置的利率计息；到期或提前取出时，按规则结算本金和利息，返还到玩家钱包。

这不是物品临时寄存——它是一个带规则的「储蓄玩法」，专门用于设计长线留存类活动。

## 解决什么问题

「存货币，到期多返」是游戏里提升留存的经典玩法，但实现起来有几个难点：

- 计息逻辑复杂（周期利率、提前支取罚金、分段利率）
- 结算要原子完成，防止并发下「取了但没到账」
- 要展示玩家「当前预计能拿多少」的实时利息预览
- 多个箱子配置同时运行（不同利率、不同周期）

存储箱模块把这些标准化：配置好箱子参数，玩家存入，平台负责计息和原子结算。

## 典型使用场景

**场景 1：定期储蓄活动**
「存满 30 天，本金 × 1.3 倍返回」。运营配置一个箱子（30 天周期、30% 利率），玩家存入任意数量的金币，30 天后取出获得 1.3 倍。到期前提前取出，只返还本金（无利息或有罚金）。

**场景 2：元旦双倍活动**
「元旦期间存钻石，3 天后返还双倍」。限时开放，活动结束后箱子关闭，新玩家不能再存。已存的按规则结算。

**场景 3：风险投资玩法**
「押注活动」：玩家存入一定数量的货币，7 天后平台按照「随机倍率」（0.5x ~ 3x）返还。较高风险感知，增加游戏趣味性。

## 核心能力

- **灵活利率**：年化利率、周期利率，支持分段利率配置
- **提前支取规则**：可设罚金（提前取出损失部分利息），可设禁止提前支取
- **实时预览**：玩家查询时实时计算「如果现在取出能拿多少」，展示透明
- **原子结算**：取款在单条 SQL 里完成，本金和利息同时到账，无并发风险
- **多箱并存**：不同利率、不同周期的箱子可以同时运行，玩家可分别存入

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 留存手段多样化，定期储蓄活动能显著拉长玩家活跃周期 |
| 产品经理 | 灵活配置存款规则和利率，快速验证不同留存玩法的效果 |
| 运营人员 | 管理台查看存款量、到期分布，提前预判结算峰值 |
| 研发团队 | 接入简单，不用自己实现计息公式和原子结算逻辑 |
