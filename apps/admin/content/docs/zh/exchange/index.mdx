兑换系统提供「花 A 得到 B」的通用兑换能力。无论是「100 积分换一个道具箱」、「活动代币换稀有皮肤」还是「多余物品换通用货币」，都可以通过配置兑换选项来实现，无需写代码。

## 解决什么问题

运营活动里几乎必有兑换环节，常见的困境是：

- 每次做活动，研发都要为兑换逻辑单独开发接口
- 限量兑换（全服只有 1000 个）难以保证并发安全
- 单用户兑换次数超限，需要自己加防刷逻辑
- 兑换没有幂等保护，网络重试导致重复发奖

兑换系统把这些问题标准化：**配置化的兑换规则 + 双层限额 + 幂等键**，运营配置好就能上，研发接一次接口后续不用改。

## 典型使用场景

**场景 1：积分商城**
活动期间玩家累计积分，活动结束时开放「积分商城」兑换。每个选项设定积分花费和对应奖励，每人限兑 3 次，全服共 5000 份。平台自动控制限额，不需要业务侧自己做。

**场景 2：节日限量礼包**
春节活动：「100 活动币 → 稀有皮肤」，全服限量 1000 个，先到先得。平台原子地扣除活动币并记录全局计数，不会超发。

**场景 3：剩余物品回收**
多余的消耗品可以换回通用货币，每人无限次但每次只能换 1 个，防止批量清仓。

## 核心能力

- **兑换配置**：一个活动配置下挂多个兑换选项，方便按活动管理
- **双层限额**：单用户限额 + 全服限额，两者独立控制
- **原子执行**：扣费和发奖在一个事务里完成，不存在半执行
- **幂等保护**：传入幂等键，重复请求不会重复执行
- **实时状态**：可查询某用户对某选项的已兑换次数

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 限量活动的库存被精确控制，不会超发造成损失 |
| 产品经理 | 不用每次写需求文档等研发排期，管理台自助配置兑换规则 |
| 运营人员 | 实时查看全局已兑换数量，掌握活动进度 |
| 研发团队 | 一次接入，后续所有兑换玩法通过配置实现，无需重复开发 |

## 与其他模块的关系
