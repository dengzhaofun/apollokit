邀请模块实现「老带新拿奖励」：每个玩家有专属邀请码，新玩家注册时带上，双方绑定邀请关系；新玩家达到某些条件触发 `qualified_at`，邀请人拿奖励。

## 数据模型

- **invite_settings**（每 org 一行）：奖励配置、资格条件、是否要求双方存在、邀请码长度规则。
- **invite_codes**：每玩家的邀请码（可轮换），`(orgId, code)` 联合唯一。
- **invite_relationships**：inviter → invitee 绑定，含 `bound_at` / `qualified_at` / `reward_claimed_at`。

## 管理端 API

```http
PUT /api/invite/settings
GET /api/invite/settings
GET /api/invite/relationships              查绑定与资格
```

## 客户端 API

```http
GET  /api/invite/end-user/my-code
POST /api/invite/end-user/rotate-code
POST /api/invite/end-user/redeem           body: { code }
POST /api/invite/end-user/mark-qualified   业务侧打 qualified 标
POST /api/invite/end-user/claim-reward     邀请人领奖

GET  /api/invite/end-user/my-invitees
```

## 业务规则

- **绑定一次性**：一个 invitee 只能绑定一个 inviter，`redeem` 幂等，重复兑换同一 code 返回已绑定。
- **自邀防御**：不允许给自己的 code 做 redeem。
- **资格门槛**：`qualified_at` 由业务方在满足条件时主动调用标记（或接事件触发）。没标记，邀请人领不到奖。
- **奖励幂等**：`claim-reward` 同一关系只能领一次，服务层用原子 `UPDATE ... WHERE reward_claimed_at IS NULL RETURNING` 防并发。
- **code 轮换**：玩家主动换 code 后，旧 code 立即失效但已绑定关系保留。

## 与其他模块组合

- [好友](../friend)：邀请成功后自动加好友（可选）。
- [物品系统](../item) / [货币](../currency)：奖励发放走这两个模块。
- [任务系统](../task)：把「邀请 N 人达资格」做成任务。
- [邮件](../mail)：资格达成时给邀请人发邮件提醒去领奖。

## 事件

`invite.bound` / `invite.qualified` / `invite.reward_claimed`。
