邀请模块实现「老带新拿奖励」：每个玩家有专属邀请码，新玩家注册时带上，双方绑定邀请关系；新玩家达成资格条件（如付费、升到 N 级）后，邀请人可领取奖励。

## 解决什么问题

老带新是拉新成本最低的渠道之一，但自己实现需要处理很多细节：

- 邀请码生成与唯一性保证
- 一个新玩家只能被一个老玩家邀请（防止重复绑定）
- 资格条件何时触发、由谁触发
- 领奖的幂等性（防止被领多次）
- 防自邀（玩家不能邀请自己）

邀请模块把这些全部封装好，业务只需在新玩家满足条件时调一个接口标记「已达资格」。

## 典型使用场景

**场景 1：付费达标领奖**
新玩家首次充值后，邀请方可领取奖励。业务在充值成功时调用「标记资格」接口，邀请人到「邀请奖励」页面一键领取。

**场景 2：等级达标领奖**
新玩家升到 10 级后，邀请关系激活，邀请人领奖。适合希望新玩家真正玩进去才给奖励的设计。

**场景 3：邀请 N 人做任务**
「邀请 5 个好友注册」作为活动任务配置，任务系统和邀请模块联动，新玩家绑定时自动推进邀请方的任务进度。

**场景 4：裂变活动专属码**
活动期间生成限定邀请码，活动结束后将专属码标记无效，不影响永久码。

## 核心能力

- **专属邀请码**：每个玩家有持久邀请码，可主动轮换，旧码立即失效
- **唯一绑定**：一个新玩家只能绑定一个邀请人，重复兑换同一码不重复绑定
- **资格门控**：新玩家达成条件后，业务侧标记「已达资格」，邀请人才能领奖
- **幂等领奖**：同一邀请关系只能领一次，并发安全
- **防自邀**：不允许玩家兑换自己的邀请码

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 老带新是获客成本最低的渠道，有效的邀请机制可以显著降低获客成本 |
| 产品经理 | 灵活配置资格条件和奖励类型，不同阶段可快速调整策略 |
| 运营人员 | 查看邀请关系数据，分析哪些渠道带来的新玩家质量更高 |
| 研发团队 | 接入成本低：仅需在资格达成时调一个标记接口 |

## 与其他模块的关系

- **[好友](../friend)**：邀请绑定成功后可选自动添加为好友
- **[物品系统](../item)** / **[货币](../currency)**：邀请奖励通过物品或货币模块发放
- **[任务系统](../task)**：将「邀请 N 人」配置为任务目标，绑定事件驱动任务进度
- **[邮件](../mail)**：资格达成时，给邀请人发邮件提醒去领奖
