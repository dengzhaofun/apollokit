跳转模块是一个**纯值对象**（value object）：它定义「一个可执行的跳转动作」的类型结构和校验规则，被 banner、对话、公告、邮件等多个模块内嵌使用，本身不存储数据、不对外暴露 HTTP 端点。

## 解决什么问题

游戏内有很多「点击后跳到某个地方」的场景：banner 点击跳活动、公告里的 CTA 跳商城、对话选项跳剧情……如果每个模块各自定义跳转结构，就会出现：

- 三个模块三种字段格式，客户端要写三段处理逻辑
- 新增一种跳转目标类型（如「跳到好友页面」），需要改三个模块
- 字符串格式的路由 key 没有类型检查，错打了字也不报错

跳转模块提供一个统一的 `LinkAction` 类型，所有嵌入点共享，客户端 exhaustive switch 一次写完。

## 支持的跳转类型

| 类型 | 说明 |
|------|------|
| `none` | 不跳，仅展示（没有 CTA） |
| `external` | 跳外链（完整 URL） |
| `internal` | 跳站内路由（注册表中的合法路由 key） |

`internal` 的路由 key 必须是预先在 `link-registry.ts` 注册过的，注册时声明参数形状。填未注册的 key 会在校验时被拒绝。

## 典型嵌入点

| 出现位置 | 用途 |
|------|------|
| [轮播](../banner) 的 `action` | 玩家点轮播图跳哪里 |
| [公告](../announcement) 的 `cta` | 公告的「行动召唤」按钮 |
| [对话](../dialogue) 的 option action | 对话选项的跳转目标 |
| [邮件](../mail) 的附件按钮 | 邮件里的跳转按钮 |

## 为什么独立成模块

- **一处改，多处跟**：新增 internal route 只改 `link-registry.ts`，所有嵌入点的校验自动更新
- **类型安全**：前后端共享 `LinkAction` 类型，客户端可以 exhaustive switch 处理所有分支
- **避免漂移**：各模块不用各自定义跳转格式，消除字段名不一致

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 产品经理 | 理解平台内所有可跳转目标的统一清单，规划 CTA 时有据可查 |
| 研发团队 | 新增跳转目标类型只改一处；客户端处理逻辑写一次 |

## 没有 API

这个模块不对外暴露 HTTP 端点。想新增 internal route，在 `link-registry.ts` 里加一条常量，然后在客户端实现对应的跳转行为。
