公告模块让你给玩家展示「橱窗式」的重要信息——停服预告、版本更新、活动开始通知。支持弹窗（首次登录强提示）和列表（随时可查）两种形态，支持按玩家属性定向展示，并记录曝光和点击数据。

## 解决什么问题

游戏运营里有大量需要主动告知玩家的信息，但发送渠道往往很分散：

- 停服维护要发个弹窗，开发者自己在代码里写死时间
- 新活动预告要在多个地方分别处理，还得记得改
- 只想给新手看的引导公告，对老玩家也显示，体验不好

公告模块统一管理所有此类信息：**管理台创建公告，客户端拉取并展示，支持定向条件和时间窗口，无需改代码**。

## 典型使用场景

**场景 1：停服维护公告**
「服务器将于今晚 22:00-24:00 维护，维护期间无法登录」。创建弹窗公告，所有玩家登录时强提示，维护结束后下架。整个过程运营自助完成，不用研发介入。

**场景 2：活动预热**
「双十一活动即将开始，11月11日0点准时上线！」。7 天前创建，设定显示时间窗口，到时间自动展示，活动正式开始后改为列表形式（从强提示退为可查）。

**场景 3：定向新手引导**
「第一次进游戏？点击查看新手教程」。只对创角 7 天内的玩家显示，老玩家不打扰。通过定向条件精准控制受众。

**场景 4：版本更新日志**
每次大版本更新发一条公告，列举更新内容。玩家在大厅公告列表里随时查看，不需要弹窗强提示。

## 核心能力

- **两种展示形态**：弹窗（强提示，首次登录弹出）+ 列表条目（可随时查看）
- **定向投放**：按等级区间、账号注册时间、标签、活动参与状态等条件过滤受众
- **时间窗口**：设定展示的开始和结束时间，自动上下架
- **CTA 跳转**：可附加按钮，点击跳转到活动、商品或外链
- **曝光/点击埋点**：统计有多少玩家看到、点击了公告，辅助运营决策

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 运营消息有统一管理入口，不用每次都找研发 |
| 产品经理 | 灵活配置展示条件，精准触达目标受众 |
| 运营人员 | 管理台全程自助：创建、预览、发布、下架、查数据 |
| 研发团队 | 接入一次，后续所有公告内容通过配置管理 |

## 与其他模块的关系

- **[Banner](../banner)**：公告是详情页（文字为主），Banner 是门面海报（图片为主），通常搭配使用
- **[邮件](../mail)**：公告没有附件，需要带奖励的告知请用邮件
- **[活动](../activity)**：活动预热用公告，活动开始后玩家参与状态通过邮件通知
