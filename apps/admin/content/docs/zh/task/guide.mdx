任务模块把玩家的一切可量化行为（击杀数、累计登录、消费次数、分享次数……）变成可配置的奖励管道。核心心智：**业务侧只上报事件，任务模块自己决定谁完成了什么**。

## 数据模型

- **category（分类）**：日常 / 周常 / 成就 / 活动任务等。分组显示用。
- **definition（定义）**：一条任务的全部规则——事件名、字段过滤器、达成值、重置周期、奖励、多级阶梯。
- **assignment（分配）**：定义 + endUserId 对，表示「这个玩家在做这个任务」。可以手动分配也可以全量默认分配。
- **progress**：每玩家 × 每任务的进度值，自动按事件累计。

## 管理端 API

```http
# 分类
POST   /api/task/categories
GET    /api/task/categories
GET    /api/task/categories/{id}
PATCH  /api/task/categories/{id}
DELETE /api/task/categories/{id}

# 定义
POST   /api/task/definitions
GET    /api/task/definitions
GET    /api/task/definitions/{key}
PATCH  /api/task/definitions/{key}
DELETE /api/task/definitions/{key}     级联清进度

# 分配
POST   /api/task/definitions/{key}/assignments
DELETE /api/task/definitions/{key}/assignments/{endUserId}
GET    /api/task/definitions/{key}/assignments
GET    /api/task/assignments                        跨任务的客服视图
```

## 客户端 API

```http
POST /api/client/task/events          上报业务事件（驱动进度）
POST /api/client/task/list            列出任务 + 进度
POST /api/client/task/claim/{taskId}  领单阶段奖励
POST /api/client/task/claim-tier      领阶段性奖励中的某一 tier
```

## 事件驱动进度

这是模块的关键机制：

1. 业务侧在玩家做动作时调 `/api/client/task/events`：
   ```json
   { "name": "kill_boss", "value": 1, "meta": { "bossId": "dragon" } }
   ```
2. 任务模块查所有 `eventName == 'kill_boss'` 的 definition，按每条的 `filter`（例如 `meta.bossId == 'dragon'`）决定是否累计到该玩家对该任务的进度。
3. 进度达到 `target` 时，`completedAt` 被写入；玩家可 claim 奖励。

过滤器支持结构化字段（`filter-fields.ts`），不是字符串匹配——这样定义可读性好，运营能在管理台配出复杂条件。

## 阶梯奖励（claim-tier）

一个任务可以配多级阈值：

```
30 次 → 领第 1 档（100 金币）
100 次 → 领第 2 档（500 金币 + 铜宝箱）
500 次 → 领第 3 档（5000 金币 + 金宝箱）
```

每档独立领取。前端在进度达到阈值时提示「可领」。

## 与活动对接

`event-forwarder.ts` 把任务完成事件转发到 [activity](../activity) 的积分系统：

- 任务完成 → 活动积分 += N → 里程碑解锁

等于让任务和活动积分联动，不需要在业务里写两遍。

## 事件产出（供 webhook）

- `task.progress`：每次进度变化（频次高，可关订阅）
- `task.completed`：任务完成
- `task.claimed`：奖励领取

`apps/server/src/modules/task/index.ts` 里声明了 event schema（`organizationId`、`endUserId`、`taskId`、`taskAlias`、`progressValue`、`completedAt`）。

## 重置周期

- `daily`：每天 UTC 0 点（或配置的本地时区 0 点）重置进度
- `weekly`：每周一
- `none`：永不重置（成就类）

重置只清 `progressValue`，不清 claim 状态——如果想让 daily 奖励每天都能领，需要在 definition 上同时勾上「重置领取状态」。
