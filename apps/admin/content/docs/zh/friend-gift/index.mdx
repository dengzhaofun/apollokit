好友赠礼模块实现「老带新赠礼 / 好友日常互送友情点」的玩法：管理员预设礼包种类，玩家在每日上限内互相赠送，收方一键领取并获得奖励。

## 解决什么问题

游戏里的好友关系如果没有互动载体，很快就变成空的数字。好友赠礼是成本最低的社交黏性机制之一：

- 玩家每天有理由主动打开游戏（领礼物、给好友送礼）
- 赠礼成本在游戏内，不涉及真实货币，运营风险低
- 但自己实现要处理：每日计数器、礼物状态流、领取发奖原子性、过期清理……

好友赠礼模块把这些都封装好，配置礼包内容，接口即用。

## 典型使用场景

**场景 1：每日友情点互送**
好友之间每天可互送一次友情点，积累后可用于兑换道具。玩家每天登录优先领礼物、给好友送礼，形成日活驱动。

**场景 2：赠礼催活**
配置"赠礼满 7 天发双倍"，激励玩家连续 7 天送礼，配合签到形成复合留存机制。

**场景 3：赛季专属礼包**
活动期间临时增加一种赠礼包，内含赛季专属道具，活动结束后将该礼包下架，全流程无需改代码。

**场景 4：礼物过期提醒**
未领取的礼物在 TTL 内自动过期，可通过事件触发推送提醒玩家及时领取，提升领取率。

## 核心能力

- **礼包配置**：管理台配置可送礼包模板（内容物、冷却时间），随时增删
- **每日上限**：发送方和接收方分别设置每日上限，双侧控制
- **状态流转**：礼物有 pending → claimed / expired / cancelled 完整状态
- **自动过期**：未领取的礼物按配置 TTL 到期自动清除
- **原子发奖**：领取操作原子地读礼包内容、调物品服务发放、标记完成

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 低成本提升日活和好友互动频率 |
| 产品经理 | 灵活配置礼包内容和每日上限，快速调整留存策略 |
| 运营人员 | 通过礼包配置推特定活动道具，不需要研发介入 |
| 研发团队 | API 一次接入，不用自己处理计数器与并发安全 |

## 与其他模块的关系

- **[好友](../friend)**：赠礼前提是双方为好友关系（可选配置）
- **[物品系统](../item)**：礼包内容通过物品模块发放至玩家背包
- **[邮件](../mail)**：玩家未上线时，可将未领礼物转邮件通知（可选）
