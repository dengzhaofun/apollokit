活动模块是运营活动的**编排引擎**。无论是「登录送礼」「累计签到领大奖」还是「完成任务积分换奖励」，都可以用一个活动来统一管理。活动有状态机（草稿→上线→归档）、可挂子玩法、可定时推进、可发活动积分和里程碑奖励。

## 解决什么问题

每次做运营活动，研发都要重新写：活动上下线逻辑、玩家参与状态、积分计算、奖励领取……而且不同活动的逻辑往往有 80% 是相同的，只有 20% 是差异化配置。

活动模块把这 80% 抽出来：**一个活动就是一个状态机 + 一套积分/里程碑体系**，差异化的玩法通过挂载子节点（node）来实现，而不是每次重写活动框架。

## 典型使用场景

**场景 1：累计签到活动**
「双十一活动：签到 7 天送限定皮肤」。创建活动，挂签到子节点，配置 7 天里程碑奖励（限定皮肤）。配置时间触发器，活动到期自动归档。整套流程无需研发写新代码。

**场景 2：积分兑换活动**
「限时积分商城：完成各类任务赚积分，积分换奖励」。活动主体管积分，各任务完成时调 `add-points` 累积，玩家攒够积分后在兑换模块消耗——活动和兑换联动。

**场景 3：多节点活动**
「新手礼包活动：首次登录 + 完成教程 + 首次充值」，每个节点是一个子任务，全部完成触发大额里程碑奖励。多节点结构让复杂活动可以清晰拆解，不用写 if-else 堆逻辑。

**场景 4：赛季活动**
每月一个赛季活动，月末归档，下月创建新的。历史赛季数据保留，可以查玩家在历史赛季的参与记录。

## 核心能力

- **状态机**：草稿（只管理端可见）→ 上线（玩家可见可参与）→ 归档（玩家不可见，数据保留）
- **子节点（node）**：将签到、任务、抽奖等挂为活动的子玩法，统一管理
- **定时推进**：通过时间触发器自动开始、切换阶段、结算，无需人工操作
- **玩家积分**：每个玩家有活动专属积分，可叠加、可查询
- **里程碑**：积分达到阈值解锁奖励，玩家主动领取，可配多档
- **玩家视角 API**：一个接口返回玩家在某活动里的完整状态（已参与、积分、里程碑）

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 活动可以快速配置上线，不依赖研发排期，加快运营节奏 |
| 产品经理 | 统一的活动框架，所有运营玩法都在同一套模型里，易于规划和复盘 |
| 运营人员 | 管理台配置活动、挂节点、看参与人数和积分分布 |
| 研发团队 | 不用为每次活动写框架代码，接入一次后续复用 |

## 与其他模块的关系

- **[签到](../check-in)** / **[抽奖](../lottery)**：作为活动子节点挂载
- **[任务](../task)**：任务进度可转发为活动积分
- **[商店](../shop)**：活动限定商品可绑定活动，活动结束时自动下架
- **[物品](../item)** / **[货币](../currency)** / **[邮件](../mail)**：里程碑奖励的实际发放渠道
