对话模块提供「剧情树编排 + 运行时推进」：策划在管理台配置节点和选项，玩家在客户端按选项走分支，每个节点可以配置奖励。整个剧情树是数据，不是代码。

## 解决什么问题

游戏中的对话和剧情如果硬编码在客户端，就会面临：

- 修改一句对话台词需要发版
- 分支剧情的状态管理（玩家走到哪了、拿了哪些奖励）要自己实现
- 剧情选项的前置条件（需要达到特定等级才解锁）每个地方都要写一遍

对话模块把剧情树完整存在服务端，玩家进度实时同步，策划可以在管理台随时修改台词和节点结构，不需要发版。

## 典型使用场景

**场景 1：主线剧情推进**
游戏主线有 100 个对话节点，玩家每次进对话拿到当前节点文本、可选选项，选择后推进到下一节点。节点进入时自动发放道具或货币奖励。

**场景 2：分支选择影响结局**
对话节点的选项可以设置前置条件（持有某道具、完成某任务），满足条件才可选。不同选择通向不同节点，产生不同结局。

**场景 3：NPC 日常对话**
每天跟特定 NPC 对话一次领奖励，对话树简单（1 个节点），`autoClaim` 奖励，节点进入即发放，无需玩家二次确认。

**场景 4：活动引导对话**
活动开始时弹出引导对话，解释活动规则。活动结束后将该对话树归档，不影响其他剧情。

## 核心能力

- **节点树配置**：管理台图形化编排节点、选项、跳转关系
- **分支条件**：选项可设前置条件（等级、道具持有、任务状态），不满足则不可选
- **节点奖励**：每个节点进入时可自动发放物品/货币奖励，幂等防重发
- **进度持久化**：玩家的对话进度在服务端持久保存，多端同步
- **重置机制**：支持重置对话进度（历史奖励记录不清）

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 剧情内容与游戏逻辑解耦，策划自主修改，无需开发 |
| 产品经理 | 支持复杂分支结构，灵活配置选项条件和节点奖励 |
| 策划 / 运营 | 在管理台直接编辑对话树，实时生效，不用等发版 |
| 研发团队 | 实现「向服务端问下一个节点」的标准交互即可 |
