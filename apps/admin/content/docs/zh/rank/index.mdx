天梯模块面向「有对局、要排位」的竞技玩法：每场对局结束后上报结果，服务端自动更新玩家的 Elo 分和段位，支持段位保护、连胜奖励和跨赛季重置。

## 解决什么问题

竞技排位是对局游戏的核心玩法，但实现一套完整的天梯系统工作量很大：

- Elo / Glicko2 等评分算法的实现与参数调优
- 段位与 Elo 分的映射关系，升段/降段的边界处理
- 连胜奖励、段位保护盾等防流失机制
- 赛季结束时的压段逻辑和赛季奖励发放
- 结算幂等：同一场对局的结果只能计入一次

天梯模块把这些封装好，业务只需要上报「谁赢了谁输了」，段位和积分变动自动完成。

## 典型使用场景

**场景 1：1v1 竞技匹配**
玩家进入匹配池，对局结束后业务上报结果，天梯模块更新双方 Elo 分，触发段位升降，展示新段位和分数变化。

**场景 2：5v5 团队排位**
多人对局，所有参与者的结果一次性上报，服务端批量计算每位玩家的新分数，保证整场结算原子完成。

**场景 3：赛季机制**
每季度重置一次，赛季结束时按规则压段（高段位玩家被压到中段位防止开局就无聊），给达到特定段位的玩家发赛季奖励。

**场景 4：段位保护**
新晋段位的玩家获得保护盾，第一次失败不掉段，让玩家有时间适应新段位，减少因段位下滑导致的流失。

## 核心能力

- **评分算法**：内置 Elo / Glicko2 可选，参数可调
- **段位系统**：自定义段位结构（几段、几小段、每段分值区间）
- **保护机制**：连胜额外加分、段位保护盾免降段
- **赛季管理**：开启/关闭赛季，结算时自动生成段位快照
- **结算幂等**：按 `externalMatchId` 去重，同一场对局只计算一次
- **完整历史**：每场对局原始记录可查，支持还原任意历史状态

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 竞技排位是留住硬核玩家的核心机制，赛季感强化持续游玩动力 |
| 产品经理 | 灵活配置段位结构和赛季规则，快速响应平衡性调整需求 |
| 运营人员 | 查看段位分布、赛季数据，设计赛季奖励策略 |
| 研发团队 | 上报对局结果即可，评分计算和段位变动完全由模块处理 |
