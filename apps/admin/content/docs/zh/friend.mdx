好友模块提供「加好友 / 收请求 / 拒绝 / 拉黑」一整套社交关系维护。关系双向,拉黑单向,所有上限可在每租户层面配置。

## 数据模型

- **friend_settings**(每 org 一行):`maxFriends` / `maxBlocked` / `maxRequests`、请求过期策略。
- **friend_relationships**:已确认的双向好友关系。
- **friend_requests**:pending / accepted / rejected / expired 状态流。
- **friend_blocks**:单向黑名单。

## 管理端 API

```http
PUT /api/friend/settings                    该 org 的上限/策略
GET /api/friend/settings
```

## 客户端 API

```http
POST /api/friend/end-user/request           发起加好友
POST /api/friend/end-user/request/{id}/accept
POST /api/friend/end-user/request/{id}/reject
POST /api/friend/end-user/request/{id}/cancel
DELETE /api/friend/end-user/{friendId}      删好友

POST /api/friend/end-user/block
DELETE /api/friend/end-user/block/{userId}

GET /api/friend/end-user/list
GET /api/friend/end-user/requests
GET /api/friend/end-user/blocks
```

## 业务规则

- **双向关系**:accept 之后在 `friend_relationships` 写两条镜像记录,查询侧任一端都能命中,删除时一并删。
- **拉黑**:单向。A 拉黑 B,A 看不到 B 发的请求;B 仍可看到 A(视 settings 可选不对称可见)。
- **上限**:好友 / 拉黑 / 未处理请求分别受 settings 控制,超限时返回 `LIMIT_EXCEEDED`。
- **请求过期**:settings 的 `requestTtl` 到期后自动转 expired。

## 与其他模块组合

- [friend-gift](./friend-gift):好友之间送日常礼物。
- [invite](./invite):邀请注册后自动结为好友(可选)。
- [leaderboard](./leaderboard):`scope=friend` 做好友内部比拼。

## 事件

`friend.requested` / `friend.accepted` / `friend.removed` / `friend.blocked`。
