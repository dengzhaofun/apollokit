Banner 模块管理「大厅上方那一排会滚动的 banner 图」。通过**分组**把不同场景的 banner 位分开（登录页、大厅顶栏、活动中心入口），每组内的多个 banner 按权重轮播。

## 数据模型

- **group（分组）**：一个具体的「banner 位」——有 alias、位置语义、场景。客户端按 alias 拉这一组的 banner。
- **banner（单图）**：一张具体的图 + CTA 跳转 + 时间窗 + 权重 + 归属 group。
- **user_state**：玩家关闭过的 banner（如果设计了关闭按钮）记在这里，下次不再展示。

## 管理端 API

```http
# 组
GET    /api/banner/groups
POST   /api/banner/groups
GET    /api/banner/groups/{id}
PATCH  /api/banner/groups/{id}
DELETE /api/banner/groups/{id}      级联删组内 banners

# 组内 banners
GET    /api/banner/groups/{groupId}/banners
POST   /api/banner/groups/{groupId}/banners
POST   /api/banner/groups/{groupId}/banners/reorder    一次性传 ordered id 列表

# 单个 banner
GET    /api/banner/banners/{id}
PATCH  /api/banner/banners/{id}
DELETE /api/banner/banners/{id}
```

## 客户端 API

```http
GET /api/client/banner/groups/{alias}
```

按 alias 拿一个组 → 返回当前时间窗内、对该玩家可见、按权重排好的 banner 列表。客户端按顺序轮播即可。

## 排序

两种方式：

- **固定顺序**：`/reorder` 传完整的 id 列表，按顺序展示。
- **权重**：每个 banner 有 `weight`，服务端按权重做稳定排序（相同权重时按创建时间）。

通常用权重——运营不想手动排，给一个 0-100 的滑块就行。

## 时间窗

banner 有 `startAt` + `endAt`。窗外不出现。典型用法：

- 活动预热期：T-7 天创建，startAt 设为 T-3 天，endAt 设为 T+14 天，到点自动上下架。
- 双十一 banner：11.01 00:00 开 — 11.11 24:00 关，到点自动关掉，无需运营半夜手动下架。

## CTA 跳转

`ctaUrl` 是一个相对路径或绝对 URL。游戏内通常是：

- `apollokit://activity/{alias}` → 跳到某活动
- `apollokit://shop/product/{alias}` → 跳到某商品
- `https://...` → 外链（新版本宣传 / 社区论坛）

协议由客户端路由解析，banner 模块不校验格式。

## 埋点

曝光 / 点击埋点复用公共事件流——建议在客户端显式上报 `banner.impression` + `banner.click`，便于分析各 banner 位的 CTR。

## 定向（可选）

与 [announcement](../announcement) 类似，banner 支持按玩家属性（等级、标签、活动状态）过滤。不设就是全员可见。

## 与其他模块

- [announcement](../announcement)：banner 是「门面海报」，公告是「详情页」。常搭配：banner 图吸引点击 → CTA 跳到公告 → 公告正文里的按钮再跳具体活动。
- [shop](../shop)：商品首屏通常就是几个 banner + 推荐列表。
