轮播模块管理游戏大厅、登录页等各场景的「banner 图位」。通过分组把不同场景的轮播位分开，每组内多张 banner 按权重轮播，带时间窗自动上下架，运营自助完成，无需改代码。

## 解决什么问题

游戏首页的视觉焦点图是最贵的运营位置，但管理起来却很痛：

- 新活动上线前，开发者要手动改代码或后台写死图片地址
- 活动结束后，运营要提醒开发者撤掉 banner，经常忘
- 多个 banner 同时存在时，排期和权重全靠口头协调

轮播模块让运营**完全自助**：在管理台上传图、设时间、配权重，到点自动出现，活动结束自动消失。

## 典型使用场景

**场景 1：活动预热 banner**
双十一前一周在首页顶部放预热海报，`startAt = 11/04 00:00`，`endAt = 11/11 24:00`——无需运营半夜手动下架。

**场景 2：多场景 banner 位分开管理**
登录页有自己的 banner 组，大厅顶栏有自己的 banner 组，活动中心入口又是一组。各组独立配置，互不干扰。

**场景 3：多 banner 权重轮播**
同时有「新版本宣传」「付费促销」「联动活动」三张 banner，按权重决定出现频次，不需要手动排班。

**场景 4：定向 banner**
高等级玩家看到「资深玩家专属礼包」banner，新玩家看到「新手引导」banner——同一个 banner 位，不同受众看到不同内容。

## 核心能力

- **分组管理**：不同场景的 banner 位用独立分组，客户端按分组 alias 拉取
- **时间窗**：每张 banner 有独立的开始/结束时间，到点自动上下架
- **权重排序**：按权重决定轮播顺序，也可手动固定顺序
- **CTA 跳转**：每张 banner 可附带跳转链接，支持站内深链和外链
- **定向投放**：按玩家等级、标签、活动参与状态等条件过滤受众
- **曝光/点击埋点**：统计各 banner 位的实际 CTR，辅助决策

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 首页视觉位由运营自主管理，无需研发介入 |
| 产品经理 | 精准控制各场景的视觉焦点，支持定向展示 |
| 运营人员 | 管理台全程自助：上图、设时间、配权重、发布、查数据 |
| 研发团队 | 按 alias 拉取分组即可，后续所有内容通过配置管理 |

## 与其他模块的关系

- **[公告](../announcement)**：banner 是「门面海报」（图片为主），公告是「详情页」（文字为主）。常见组合：banner 吸引点击 → CTA 跳到公告正文 → 公告正文里的按钮再跳具体活动
- **[商店](../shop)**：商品首屏通常是几张 banner + 推荐列表，两者搭配使用
- **[活动](../activity)**：活动期间专属 banner，活动结束后通过时间窗自动撤掉
