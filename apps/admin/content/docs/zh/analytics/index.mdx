埋点分析模块是 ApolloKit 与分析引擎（Tinybird）的对接层。ApolloKit 的事件数据通过事件中心产生，分析模块负责**为租户签发可以直接查询分析数据的 scoped token**，让管理台看板不经过自己的后端就能实时拉取指标。

## 解决什么问题

游戏运营需要数据，但传统的数据架构有延迟：

- 埋点数据进数据仓库 → 跑 ETL → 出报表，从事件发生到看到数据可能有几个小时
- 管理台看板要查数据，还需要自己的后端 API 做中转
- 多租户场景下，每个租户只能看自己的数据，越权访问是安全隐患

分析模块用 scoped token 机制解决这些问题：前端直接带 token 查询分析引擎，token 里锁死 org_id，物理上不可能越权。

## 典型使用场景

**场景 1：DAU 实时看板**
管理台首页展示今日 DAU、昨日对比、本周趋势。看板直接带分析 token 调 Tinybird pipe，数据实时刷新，无需等待 ETL。

**场景 2：活动转化率分析**
活动期间实时查看公告曝光数、CTA 点击率、活动参与人数，辅助运营决策是否需要追加推送。

**场景 3：任务完成率监控**
新上线的任务体系，实时看哪些任务完成率过低（可能太难）、哪些过高（可能太简单），快速调整奖励配置。

**场景 4：赛季数据汇报**
赛季结束后，管理台一键生成赛季报告：新增、活跃、付费、各榜排名分布——数据直接来自分析引擎，不需要研发写专用接口。

## 核心能力

- **scoped JWT 签发**：token 内嵌 org_id，Tinybird pipe 用参数锁死，物理上无法跨租户越权
- **短 TTL**：默认 10 分钟，前端定期刷新即可保持长会话
- **无中转**：前端直接调分析引擎，延迟最低
- **不做 OLAP**：ApolloKit 不自建分析引擎，插件式对接，可切换后端

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 实时数据看板，决策有依据 |
| 产品经理 | 快速验证新功能的数据效果，不用等每日报表 |
| 运营人员 | 活动进行中实时看数据，及时调整策略 |
| 研发团队 | 对接一次 token 签发，看板查询不经过业务后端 |
