关卡模块做「章节 → 关卡 → 星级」的结构化关卡系统:关卡有解锁规则、通关带星、满星额外奖励。

## 数据模型

- **level_configs**(每 org 一份):一套关卡体系的总参数(默认最大星数、全局解锁策略)。
- **level_stages**:章节(第一章、第二章...),含图标、描述、解锁条件。
- **levels**:章节内的单个关卡,含最大星数、每星的奖励、解锁规则(auto / level_clear / stars_total / stage_clear)。
- **level_user_progress**:玩家 × 关卡的进度,含 cleared / starsEarned / lastClearedAt。

## 管理端 API

```http
POST   /api/level/configs
POST   /api/level/stages
POST   /api/level/stages/{id}/levels
PATCH  /api/level/levels/{id}
DELETE /api/level/levels/{id}
```

## 客户端 API

```http
POST /api/level/end-user/progress          body: { levelId, stars }    上报通关
GET  /api/level/end-user/stages            章节+关卡+进度一把拉
POST /api/level/end-user/claim-star-reward body: { levelId, starIndex }
```

## 业务规则

- **解锁规则(discriminated union)**:
  - `auto`:始终开放。
  - `level_clear`:过 levelId 才开。
  - `stars`:累计星数达 N 才开。
  - `stage_clear`:上一个章节所有关卡通关。
- **星级锁定**:本关曾经 3 星过后,后续拿 2 星不会降星(`MAX` 语义)。
- **奖励领取**:每颗星的奖励独立,`claim-star-reward` 按 `(userId, levelId, starIndex)` 幂等,重复领不会翻倍。
