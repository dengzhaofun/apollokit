> ⚠️ 本页讲的是 **游戏客户端 SDK** `@apollokit/client`——给接入方游戏 / 应用直接调用 `/api/client/*` 的那一层。
>
> 要构建**管理端工具**(内部后台 / 脚本 / CI),请看 [sdk](./sdk)(`@apollokit/admin`)。

`@apollokit/client` 是从 ApolloKit 的 OpenAPI 规范自动生成的 TypeScript SDK,包装了:

- `cpk_` publishable key 的 `x-api-key` 头注入
- 可选的 HMAC-SHA256 签名(服务端模式)
- 所有 `/api/client/*` 端点的类型安全调用方法

底层用 `@hey-api/client-fetch`,运行时只依赖 Web Standards(`fetch` + Web Crypto),**浏览器 / Node 18+ / Workers / Deno / Cocos Creator 都能跑**。

## 安装

```bash
pnpm add @apollokit/client
```

## 初始化

### 服务端模式(推荐所有非浏览器环境)

服务器持有 `secret`,SDK 自动算 HMAC:

```ts
import { createClient } from '@apollokit/client';

const client = createClient({
  baseUrl: 'https://api.yourgame.com',
  publishableKey: 'cpk_XXXXXXXX',
  secret: process.env.APOLLOKIT_SECRET, // csk_XXXX
});
```

### 浏览器模式(签名必须预先在服务端算好)

```ts
// 服务端接口返回预签名的 userHash
async function getUserHash(endUserId: string): Promise<string> {
  const res = await fetch('/my-server/apollokit-sign', {
    method: 'POST',
    body: JSON.stringify({ endUserId }),
  });
  return (await res.json()).userHash;
}

// 客户端只持 publishableKey
import { createClient } from '@apollokit/client';
const client = createClient({
  baseUrl: 'https://api.yourgame.com',
  publishableKey: 'cpk_XXXXXXXX',
  // 注意:不传 secret
});
```

> 永远不要把 `csk_*` 打进前端 bundle。HMAC 签名的整个意义就是不让前端持有 secret。

## 端到端示例:签到

```ts
import { createClient, signEndUser, postCheckIns } from '@apollokit/client';

const client = createClient({
  baseUrl: 'https://api.yourgame.com',
  publishableKey: 'cpk_XXXXXXXX',
  secret: process.env.APOLLOKIT_SECRET,
});

async function dailyCheckIn(playerId: string) {
  // 服务端模式:SDK 没有自动注入 user-hash(需要每用户不同),你得手工算一下
  const userHash = await signEndUser(playerId, process.env.APOLLOKIT_SECRET!);

  const { data, error } = await postCheckIns({
    headers: {
      'x-end-user-id': playerId,
      'x-user-hash': userHash,
    },
    body: { configKey: 'daily' },
  });

  if (error) {
    console.error('check-in failed', error.code, error.requestId);
    return;
  }
  return data; // { alreadyCheckedIn, state, target, remaining, ... }
}
```

## 签名工具

`signEndUser` 是 HMAC-SHA256 的薄包装:

```ts
import { signEndUser, computeHmac } from '@apollokit/client';

// 语义化别名
const hash = await signEndUser('player_42', 'csk_XXX');
// 等价于
const hash2 = await computeHmac('player_42', 'csk_XXX');
```

双向结果相同,二选一。

## 类型重新生成

SDK 带生成脚本。服务端 `/openapi.json` 变了之后,在 monorepo 里:

```bash
pnpm --filter=@apollokit/client generate
```

会重新拉 OpenAPI 并把 `src/generated/*.gen.ts` 重生成。CI 上建议加一个校验步骤,确保生成产物与源码一致。

## 已内建的错误体

SDK 的 response 形如 `{ data, error }`,其中 `error` 是 [errors](./errors) 里描述的统一错误对象:`{ error, code, requestId }`。程序里按 `error.code` 做分支处理,按 `error.requestId` 做日志关联。

## Dev 模式

凭证开 `devMode: true` 时,SDK 可以不算 HMAC——可以跳过 `x-user-hash` header。**但上线前务必关闭**,否则任何人拿到 cpk 就能冒充任意 endUserId。

## 其他语言

TypeScript 已发布。C# / Go / Python 在计划中,优先级:

1. **C#**(Unity 客户端)
2. **Go**(自建服务端代签)
3. **Python**(数据分析脚本 / 工具链)

在此之前,其他语言接入:

- 用标准 OpenAPI 代码生成工具(`openapi-generator`、`openapi-typescript` 等)消费 `/openapi.json`。
- HMAC-SHA256 算法在所有主流语言都有一行实现——参考 [authentication](./authentication#hmac) 的跨语言示例。

## 常见问题

**SDK 跟 `/openapi.json` 脱节了怎么办?**
重跑 `generate` 脚本。SDK 包里只留生成产物,手工修改会被下次覆盖。

**能不能在 Cocos Creator / Unity WebGL 里跑?**
TypeScript SDK:可以(依赖 Web Crypto)。C# SDK:规划中,会额外提供 Unity Package Manager 包。

**请求最大并发?**
没有硬限。按 org 维度做了 Upstash Redis 限流(如启用),400 QPS 软上限,可按需调高。
