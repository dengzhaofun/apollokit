邮件模块让你给玩家发「带附件的系统消息」。签到奖励、活动补偿、故障赔偿、运营公告——凡是需要「送到玩家收件箱」并且「玩家主动领取」的，都适合走邮件。

## 解决什么问题

游戏里的玩家触达有两种主要形式：公告（玩家被动看）和邮件（玩家主动收）。邮件的独特价值在于：

- **可以带附件**：实物奖励（物品、货币）通过附件随信发放，玩家领取时才到账
- **有收件记录**：玩家能回翻历史邮件，不像弹窗消失就没了
- **可以撤回**：发错了的奖励还有时间撤回，降低运营失误的影响

没有统一邮件模块时，补偿逻辑散落在各处——有时直接进背包，有时发公告，没有统一的已读/未读/已领状态，客服跟进困难。

## 典型使用场景

**场景 1：活动发奖**
活动结束后，运营选定获奖玩家列表，批量发送邮件，附件包含奖品物品和货币。玩家进游戏后看到邮件提示，主动点击领取，系统幂等地将附件发放到背包。

**场景 2：故障补偿**
服务器出现问题导致部分玩家掉线。运营在管理台发广播邮件（全服可见），附件为补偿礼包，7 天内有效。运营可实时查看有多少人已领取。

**场景 3：撤回误发**
运营发现某封邮件奖励配置有误（多发了 10 倍），在玩家大量领取之前撤回该邮件——软删除后玩家立刻看不到，但已领取记录不受影响。

**场景 4：定向玩家触达**
客服给单个玩家发送补偿邮件，带定制化的附件内容和说明文字，无需研发介入，管理台直接操作。

## 核心能力

- **三种投放模式**：广播全服 / 指定用户列表 / 单用户定向
- **附件支持**：物品和货币组合附件，玩家领取时原子发放
- **领取幂等**：重复领取请求不会重复发放奖励
- **撤回与过期**：支持软撤回（数据保留）和硬删除，支持过期时间
- **已读/已领状态**：玩家端展示，管理端可查统计

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 补偿和活动发奖有迹可查，降低运营失误风险 |
| 产品经理 | 灵活的投放策略，覆盖各类运营场景 |
| 运营人员 | 管理台直接发邮件、撤回、查领取率，不依赖研发 |
| 研发团队 | 统一的邮件 + 附件发放接口，代替各模块的「直接进背包」逻辑 |

## 与其他模块的关系

- **[物品系统](../item)** / **[货币](../currency)**：附件通过它们实际发放
- **[公告](../announcement)**：公告是橱窗式弹窗，没有附件；邮件是收件箱，有附件
- **[兑换码](../cdkey)**：兑换码兑换成功后通过邮件附件发放奖励
