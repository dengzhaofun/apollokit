公会模块实现游戏中的「大厅型社交组织」：玩家可创建或加入公会，会长/官员/普通成员角色分级，公会可升级扩容，支持审核入会和贡献追踪。

## 解决什么问题

公会是提升中长线留存最有效的机制之一，但实现一套完整的公会系统工作量很大：

- 成员权限分级（会长/官员/成员），每类权限对应不同操作
- 公会等级与经验：打怪、捐资、完成活动都可以贡献经验，贡献要原子计入
- 加入方式：直接加/申请审批/仅邀请，不同场景需要切换
- 会长离开时的继承规则，解散规则

公会模块把这些全部封装，配置好规则，接口直接调用。

## 典型使用场景

**场景 1：基础公会系统**
玩家创建公会、设置公会介绍和审核方式，其他玩家搜索后申请加入，会长审批通过即成为成员。

**场景 2：公会升级扩容**
公会等级越高，成员上限越大。成员完成日常任务贡献 exp，公会升级时所有成员的成员上限同步扩展。

**场景 3：公会限定活动**
活动期间只有公会成员才能参与，公会模块提供成员资格查询接口，活动模块据此做门禁。

**场景 4：贡献排行榜**
展示公会内贡献值最高的成员，激励成员积极参与。贡献记录有完整日志，可追溯。

## 核心能力

- **角色分级**：会长 / 官员 / 普通成员，各级权限清晰
- **审核入会**：支持直接加入、申请审批、仅邀请三种方式，可按需切换
- **升级系统**：公会有等级与经验，升级可扩展成员上限
- **贡献记录**：每次贡献有原子日志，支持查询成员贡献历史
- **转让与继承**：会长可主动转让，离队时自动触发继承规则

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 公会是中长线留存的核心机制，有公会的玩家流失率显著更低 |
| 产品经理 | 灵活配置公会规模、升级规则、加入方式，适配不同游戏类型 |
| 运营人员 | 可查看公会列表、成员构成、贡献数据，辅助活动设计 |
| 研发团队 | API 一次接入，并发写入安全（贡献与升级的原子性由模块保证） |

## 与其他模块的关系

- **[排行榜](../leaderboard)**：公会内贡献榜，`scope=guild` 按公会分榜
- **[活动](../activity)**：公会限定活动，以公会成员资格作为参与条件
- **[邮件](../mail)**：向公会全体成员发公告/奖励，以成员名单为收件人源
