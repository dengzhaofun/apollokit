> 本页面面向研发团队。产品背景与使用场景请见[产品说明](./index)。

ApolloKit 在 `/api/mcp` 暴露一个 [Model Context Protocol](https://modelcontextprotocol.io) 端点，让任何 MCP 兼容的 LLM 客户端（Claude.ai、Cursor、Claude Code、自研 AI agent 等）直接对话式访问你的 ApolloKit 项目。第一版纯只读：让 agent 能**读**文档、玩家、审计日志、数据指标——不能写。

## 能用来干嘛

- **运营**：在 Claude.ai 直接问「玩家 abc 这个月充值多少」「上周新增对比」「刚才谁改了 lottery 配置」——少了登录后台 → 找页面 → 点筛选的所有步骤。
- **客户支持**：玩家投诉「签到没拿到奖」→ 让 Claude 拉 audit log + 玩家档案 → 自动归因。
- **接入开发**：在 Cursor 写集成代码时问「ApolloKit 的 cdkey schema 是什么」→ 文档原文直接进 LLM 上下文，边查边写。
- **自动周报**：Claude 调 `query_analytics` 拉数据 + 自动排版 markdown，5 分钟出周报。
- **第三方 AI 工具**：别家做的运营 bot / Discord bot / BI 看板，连 MCP 即可，不用写 SDK 集成代码。

## 接入

### 1. 准备 apiKey

复用现有的 admin API key（`ak_…` 前缀）。在管理台「设置 → API Keys」生成或复用一把。MCP 鉴权直接走这把 key，scope 自动绑定到你的 org。

> **安全提示**：apiKey 即等同于让 LLM 访问该 org 的全部数据。第一版全只读，所以即使泄露也不会改坏数据；但仍**强烈建议**：
> - dev 环境和 prod 环境用不同的 key
> - 给 LLM 用的 key 单独签发，不要复用给其他自动化任务
> - 一旦不再使用，立即在管理台 revoke

### 2. 端点信息

```
URL:     https://<your-apollokit>/api/mcp
Method:  POST
Auth:    x-api-key: ak_…
Transport: Streamable HTTP (MCP 2025-03 spec)
```

### 3. Claude.ai 配置示例

在 Claude.ai 的「Connectors」/「Custom MCP」里添加：

```json
{
  "name": "ApolloKit",
  "transport": {
    "type": "http",
    "url": "https://<your-apollokit>/api/mcp",
    "headers": {
      "x-api-key": "ak_…"
    }
  }
}
```

### 4. Cursor / Claude Code 配置示例

`.cursor/mcp.json` 或 `~/.claude/mcp.json`：

```json
{
  "mcpServers": {
    "apollokit": {
      "url": "https://<your-apollokit>/api/mcp",
      "headers": {
        "x-api-key": "ak_…"
      }
    }
  }
}
```

## 当前可用的 tool

第一版（纯只读）共 4 个 tool：

| 名称 | 输入 | 用途 |
|---|---|---|
| `echo` | `message` | 回环测试，验证连接和鉴权 |
| `search_docs` | `query`, `locale?`, `limit?` | 全文搜索 ApolloKit 文档（中英双语） |
| `read_doc` | `path` | 读取一篇文档完整 markdown，路径形如 `zh/check-in` |
| `list_doc_modules` | — | 列出所有有文档的功能模块（按经济/运营/社交/内容/系统/集成 6 类分组） |

> 玩家查询、审计日志、数据指标 4 个只读 tool 在 Phase 2 上线。
> 写操作类 tool（发奖、群发邮件、补发兑换码等）在 Phase 3 上线。

## 验证连通

最快的方式是用 `echo` tool。在 Claude.ai 配置完后说一句：

> 用 ApolloKit 的 echo tool 回个「hello」

如果返回了字符串 `hello`，链路就通了。下一步可以问：

> 列出 ApolloKit 支持哪些功能模块

它会调 `list_doc_modules`，返回 6 大类共 30 个模块的清单。

## 故障排查

| 现象 | 原因 |
|---|---|
| `401 Unauthorized` | apiKey 缺失、错了或被 revoke |
| `406 Not Acceptable` | 客户端没在 `Accept` 里同时声明 `application/json` 和 `text/event-stream`。绝大多数 MCP 客户端会自动处理；自己写 fetch 时要加上 |
| `415 Unsupported Media Type` | `Content-Type` 不是 `application/json` |
| `search_docs` 返回 `search failed: HTTP 5xx` | 文档站（admin worker）不可达。dev 环境记得 `pnpm dev` 同时把 admin 和 server 都起起来 |
| Claude 调用 tool 但没返回结构化结果 | 检查 LLM 客户端是否支持 `structuredContent` 字段（绝大多数支持，老版本可能只读 `content[].text`）|

## 与 SDK 的关系

MCP 不是 SDK 的替代品，而是**互补**：

- **SDK**（[server SDK](../sdk/server-typescript)、[client SDK](../sdk/client-typescript)）：给程序员用，写代码集成
- **MCP**：给 LLM/AI agent 用，对话式访问

你的同一个 ApolloKit 项目可以同时跑 SDK 集成 + MCP 集成，互不干扰。
