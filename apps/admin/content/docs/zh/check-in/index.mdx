签到系统让玩家每天有理由打开游戏。支持按周/月重置、连续签到追踪、以及按签到天数配置差异化奖励。配置一次，持续运转，无需研发维护。

## 解决什么问题

「每日签到」是每款游戏都要做、每次都要重复开发的功能：

- 每天只能签一次，重复签不报错但不重复发奖
- 连续签到 7 天有额外奖励，断签了就重置计数
- 周一到周日为一个周期，新周开始重新计算
- 不同天数发不同奖励（第 1 天 100 金币，第 7 天 500 钻石）

这些规则写起来不复杂，但每个细节都需要处理，而且每次做新活动又要重写。签到系统把这一切变成**配置**：设定规则和奖励，玩家调接口，发奖自动完成。

## 典型使用场景

**场景 1：每周签到活动**
每周一重置，签到 7 天获得全勤奖。每天奖励递进：第 1-6 天发小额金币，第 7 天发稀有物品。玩家连续签才能拿满奖，有效拉动回访。

**场景 2：月度活动签到**
节日活动期间，单独配置一个签到（不影响日常签到），签满活动指定天数解锁限定奖励。活动结束后停用该配置，数据保留。

**场景 3：活动积分联动**
签到配合活动模块：签到第 N 天时，额外给玩家加活动积分，积分达到里程碑发大奖。签到系统只负责签到本身，积分联动通过事件驱动实现。

## 核心能力

- **三种重置模式**：永不重置（成就型）/ 按周重置 / 按月重置
- **连续签到追踪**：记录当前连续天数、历史最长连续天数
- **差异化奖励**：按签到第几天配置不同奖励，自动发放到背包
- **多签到配置**：可同时运行多个配置（日常签到 + 活动签到互不干扰）
- **时区支持**：按项目时区判断「今天」，适配全球玩家
- **重复签到保护**：已签到时调接口不报错，返回今日已签状态

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 每日回访是留存的基础，签到是驱动日活最低成本的手段 |
| 产品经理 | 灵活配置签到规则和奖励，快速上线不同类型的签到活动 |
| 运营人员 | 管理台配置奖励、查看每日签到人数统计 |
| 研发团队 | 一次接入，后续所有签到活动通过配置搞定，无需重复开发 |
