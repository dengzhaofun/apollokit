商店模块是玩家「花钱买东西」的标准化管道。从最简单的直购礼包，到复杂的战令/成长基金，都能通过商品配置来实现。支持时间窗口、限购、多级分类、标签筛选，以及阶段化发奖。

## 解决什么问题

自建商城是每个游戏团队都绕不过去的重复劳动：

- 每次做新礼包，研发要写购买接口、扣费逻辑、发奖逻辑
- 限购（首充只能买一次）需要自己加状态判断
- 战令/成长基金的阶段领取逻辑复杂，容易出 bug
- 上下架时间控制、库存管理、购买记录查询——每个细节都要单独处理

商店模块把这些变成**配置**：运营在管理台填好商品信息，研发接一次购买接口，之后的所有商品都不用改代码。

## 典型使用场景

**场景 1：直购礼包**
节日礼包：6.8 元，包含 100 钻石 + 3 个道具箱。配置商品、设定价格和产出，上架后玩家即可购买。限每人购买 1 次，通过 `perUserLimit: 1` 控制。

**场景 2：首充特惠**
用户注册后首次充值享双倍奖励。商品设置 `perUserLimit: 1` + 特殊奖励，平台自动判断是否首次购买，无需研发额外开发。

**场景 3：战令 / 成长基金**
玩家一次性购买战令，解锁 10 个奖励阶段，随游戏进度逐步领取。商品绑定多个 stage，每个 stage 可设置领取条件（达到某个等级、完成某个任务）。

**场景 4：每日特惠**
限时低价商品，每天刷新，每人每天只能买一次。通过上下架时间窗口 + 每人每日限购实现，运营无需手动操作。

## 核心能力

- **商品分类与标签**：支持树形分类和多标签，方便前端筛选和运营管理
- **灵活定价**：以任意货币计价，支持组合产出（多种物品 + 货币）
- **时间窗口**：商品自动按配置时间上下架，无需手动操作
- **双层限购**：每人限购 + 全服库存，两者独立控制
- **成长包/战令**：一次购买 + 分阶段领取，每个阶段可设领取条件
- **购买状态查询**：玩家端展示每个商品的可购状态（已购/限量已抢完/未到时间）

## 适合谁

| 角色 | 关注点 |
|------|--------|
| 决策者 / 老板 | 商城是核心变现路径，配置化减少研发排期，加快新商品上线速度 |
| 产品经理 | 灵活配置各类商品形态（直购、战令、首充），无需每次写开发需求 |
| 运营人员 | 管理台直接上架商品、设置价格和时间窗口，实时查看各商品购买量 |
| 研发团队 | 一次接入购买接口，后续商品全部通过配置管理，维护成本极低 |
